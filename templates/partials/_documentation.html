‚ú® What It Does

  • OpenAI API Compatibility: Accepts requests on /v1/chat/completions and /v1/models, transforming them for the Gemini API.
  • üöÄ High Performance (NEW): Async architecture with HTTP connection pooling, parallel tool execution, and smart caching delivers 3-5x speed improvement.
  • Streaming & History: Handles streaming responses with word-boundary buffering (no broken words!) and automatically truncates conversation history to fit context windows.
  • üÜï Dual Path Mode (NEW): Two distinct modes for working with code:
    • code_path= - Recursively loads all code files as text context (up to 4MB) for review and analysis. No tools, just pure code context.
    • project_path= - Activates full AI agent mode with 24 built-in development tools for navigation, analysis, modification, execution, and git operations. Perfect for actual development work.
  • Multimodal Support: Supports image, audio, and PDF file embedding directly from local paths (e.g., image_path=..., pdf_path=..., audio_path=...).
  • Built-in Development Tools: 24 tools including file operations, code search, patch application, command execution (pytest, awk, sed, grep, npm), and full git integration.
  • Function Calling: Integrates with external tools via the Multi-Tool Communication Protocol (MCP) for advanced, structured function calling. Tool selection is highly configurable: profiles (System Prompts/Overrides) can explicitly select or disable specific MCP functions/tools, overriding the default context-aware selection logic. Native Google tools (Search, URL Context) can also be enabled via profiles.
  • Image Generation: Includes a dedicated UI and API endpoint for generating images with compatible models.
  • Advanced Prompt Control: Lets you define and apply both system-level prompts and dynamic prompt overrides based on keywords.
  • Multi-Chat UI: The web interface supports multiple, persistent chat sessions, each with its own history and file storage.
  • Efficient Caching: Caches model lists, model-specific details (like token limits), tool outputs, and prompt cache entries to reduce upstream API calls.
  • Integrated Developer Toolkit: The Docker container includes a ready-to-use development environment with Node.js 22+ and the Docker CLI. This enables direct execution of `npx mcp-tools` for testing function calls and allows scripts inside the container to manage the host's Docker environment via the mounted Docker socket.

üöÄ How to Use

1. Setup and Run

Before running the server, you need to set environment variables:


 export API_KEY="YOUR_GEMINI_API_KEY"
 export UPSTREAM_URL="https://generativelanguage.googleapis.com"
 export SERVER_HOST=0.0.0.0
 export SERVER_PORT=8080
 export ASYNC_MODE=true  # Optional: enable async mode for better performance

Then, run the server:


 python run.py

or with Docker:

docker-compose up -d

The server will start on http://0.0.0.0:8080 by default.

üí° Pro Tip: Use ASYNC_MODE=true for 3-5x better performance with concurrent requests!

2. Configure JetBrains AI Assistant

To use this proxy with JetBrains IDEs:

  1. Open AI Assistant settings (Settings > Tools > AI Assistant > Models).
  2. Select the "OpenAI API" service.
  3. Set the Server URL to: http://<your-server-ip-or-localhost>:8080/v1/
  4. The model list will be fetched automatically. You can leave it as default or choose a specific one.

Note: The path must end with /v1/ because the IDE will append /chat/completions or /models to it.

üéØ Ignore Patterns (Both Modes)

Both code_path= and project_path= support custom ignore patterns:

# Ignore specific file types
code_path=. ignore_type=log|tmp|cache

# Ignore specific files
code_path=. ignore_file=*.test.js|*.spec.ts

# Ignore directories
code_path=. ignore_dir=docs|examples|legacy

# Same syntax for project_path
project_path=. ignore_type=cache

Default Ignores: .git, node_modules, __pycache__, venv, build, dist, *.pyc, *.log, minified files, lock files, and more.

üß† Context Management Settings

Control how conversation history is processed and sent to the model:

Setting Default Description
SELECTIVE_CONTEXT_ENABLED true When enabled, uses relevance scoring to intelligently select the most pertinent messages from conversation history instead of sending all messages. This helps optimize token usage and improve response quality.
CONTEXT_MIN_RELEVANCE_SCORE 0.3 Minimum relevance threshold (0.0-1.0) for including a message in context. Higher values (e.g., 0.5-0.7) mean stricter filtering, only keeping highly relevant messages. Lower values (e.g., 0.1-0.3) are more permissive.
CONTEXT_ALWAYS_KEEP_RECENT 15 Number of most recent messages to always include in context, regardless of their relevance score. This ensures continuity in the conversation even if recent messages don't score highly.
üí° Tip: These settings can be easily adjusted through the Configuration page in the Web UI under the "Context Management" section. Changes are saved to the .env file and take effect immediately.

üîí Security Settings

Control access restrictions for builtin development tools:

Setting Default Description
ALLOWED_CODE_PATHS Empty (unrestricted) Comma-separated list of root directories that builtin tools can access. When set, tools like list_files, get_file_content, create_file, execute_command, etc., will only work within these directories.

Example: /home/user/projects,/opt/workspace,/var/www

Use cases:
  • Restrict AI access to specific project directories
  • Prevent accidental access to system files
  • Create sandboxed development environments
  • Multi-user scenarios where isolation is needed

Leave empty to allow access to all paths (default behavior).
MAX_CODE_INJECTION_SIZE_KB 4096 Maximum total size in kilobytes for files loaded via code_path= (default is 4MB). This prevents excessive token usage from very large codebases or directories. Can be configured on the main page.
‚ö†Ô∏è Important: This setting affects all 24 builtin development tools. When project_path= or code_path= is used, tools will only access files within the configured allowed directories. This provides an additional security layer beyond the standard project root restriction.
